# Веб-сайт приходской школы при храме

## 1. Назначение проекта

Проект представляет собой веб-приложение для приходской школы при храме. Сайт помогает прихожанам и ученикам узнавать о кружках, мероприятиях и расписании занятий, а преподавателям и ответственным лицам заранее понимать, кто будет присутствовать на занятиях.

Сайт не должен ощущаться как строгая корпоративная система или учебный портал колледжа. Основной характер проекта: спокойный, уважительный, теплый и понятный для небольшой приходской общины.

## 2. Основные задачи

- Публикация информации о приходской школе и мероприятиях храма.
- Отображение списка кружков и занятий.
- Запись пользователей на кружки.
- Формирование персонального расписания пользователя.
- Возможность отметить присутствие или отсутствие на занятии.
- Возможность администратора управлять мероприятиями, кружками и расписанием.
- Возможность администратора видеть, кто планирует прийти на конкретное занятие.

## 3. Роли пользователей

### 3.1. Гость

Неавторизованный пользователь.

Возможности:

- просматривать главную страницу;
- просматривать список кружков;
- видеть информацию о мероприятиях;
- перейти к регистрации или входу.

Ограничения:

- не видит страницу расписания;
- не может записываться на кружки;
- не может отмечать присутствие;
- не имеет доступа к профилю.

### 3.2. Пользователь

Авторизованный пользователь сайта.

Возможности:

- просматривать главную страницу;
- просматривать кружки;
- записываться на кружки;
- видеть свое персональное расписание;
- отмечать, будет ли он присутствовать на конкретном занятии;
- отмечать присутствие или отсутствие на весь день;
- просматривать календарь расписания на месяц;
- смотреть свой профиль и список выбранных кружков.

### 3.3. Администратор

Ответственный пользователь, управляющий содержимым сайта.

Возможности:

- добавлять, редактировать и удалять мероприятия;
- добавлять, редактировать и скрывать кружки;
- создавать и редактировать расписание занятий;
- просматривать список пользователей, записанных на кружки;
- просматривать отметки присутствия по каждому занятию;
- видеть, сколько человек планирует прийти на занятие.

## 4. Страницы сайта

### 4.1. Главная

Доступна всем пользователям.

Содержит:

- краткую информацию о приходской школе;
- спокойный приветственный блок;
- список ближайших мероприятий;
- возможно, краткий блок с направлениями кружков;
- кнопку перехода к кружкам.

Мероприятие состоит из следующих полей:

- изображение;
- название;
- описание;
- дата проведения.

Администратор может добавлять, редактировать и удалять мероприятия.

### 4.2. Кружки

Доступна всем пользователям.

Для гостя:

- отображается список кружков;
- отображается описание каждого кружка;
- вместо записи предлагается войти или зарегистрироваться.

Для авторизованного пользователя:

- отображается список кружков;
- есть возможность записаться на кружок;
- видно, на какие кружки пользователь уже записан;
- можно перейти к расписанию после записи.

Кружок может содержать:

- название;
- описание;
- изображение;
- преподавателя или ответственное лицо;
- возрастную группу, если нужно;
- статус активности.

### 4.3. Расписание

Доступно только авторизованным пользователям и администраторам.

Гость не должен видеть пункт "Расписание" в навигации и не должен иметь возможность открыть страницу напрямую.

Для пользователя:

- отображается расписание на неделю;
- показываются только занятия тех кружков, на которые пользователь записан;
- у каждого занятия есть выбор "буду" / "не буду";
- по умолчанию считается, что пользователь будет присутствовать;
- рядом с днем недели можно отметить "буду" / "не буду" на весь день;
- доступен календарь на месяц;
- дни отсутствия выделяются красным или другим заметным, но не агрессивным цветом.

Для администратора:

- можно создавать занятия;
- можно редактировать дату, время, кружок, описание и место занятия;
- можно смотреть список присутствующих и отсутствующих;
- можно видеть количество людей, которые планируют прийти.

### 4.4. Профиль

Для гостя:

- отображаются кнопки "Войти" и "Зарегистрироваться".

Для пользователя:

- отображаются личные данные;
- отображается список кружков, на которые пользователь записан;
- можно посмотреть ближайшие занятия;
- возможно, можно отписаться от кружка.

Для администратора:

- профиль может дополнительно содержать переходы к управлению сайтом.

## 5. Логика кружков и расписания

Важно разделять кружок и конкретное занятие.

Кружок - это постоянное направление, например "Хор", "Лепка", "Рисование".

Занятие - это конкретная встреча по кружку в определенную дату и время.

Пользователь записывается не на отдельное занятие, а на кружок. После записи в его расписании появляются занятия этого кружка.

Пример:

- пользователь записался на кружок "Хор";
- администратор добавил занятие хора на воскресенье в 12:00;
- это занятие появилось в расписании пользователя;
- пользователь может отметить, будет он на нем или нет.

## 6. Рекомендуемая структура базы данных

### users

Пользователи сайта.

Поля:

- id;
- name;
- email;
- password_hash;
- role;
- created_at.

### events

Мероприятия на главной странице.

Поля:

- id;
- title;
- description;
- image_url;
- event_date;
- created_at.

### circles

Кружки приходской школы.

Поля:

- id;
- title;
- description;
- image_url;
- teacher_name;
- age_group;
- is_active;
- created_at.

### circle_members

Записи пользователей на кружки.

Поля:

- id;
- user_id;
- circle_id;
- created_at.

### lessons

Конкретные занятия.

Поля:

- id;
- circle_id;
- title;
- description;
- lesson_date;
- start_time;
- end_time;
- location;
- created_at.

### lesson_attendance

Отметки присутствия.

Поля:

- id;
- user_id;
- lesson_id;
- status;
- updated_at.

Статусы:

- attending;
- absent.

Если отметки нет, система может считать, что пользователь планирует присутствовать.

## 7. Основные API-разделы

Примерная структура серверной части:

- auth - регистрация, вход, выход, проверка пользователя;
- users - профиль пользователя;
- events - мероприятия;
- circles - кружки;
- circle-members - запись на кружки;
- lessons - расписание занятий;
- attendance - отметки присутствия;
- admin - административные действия, если потребуется отдельный раздел.

## 8. Визуальный стиль

Сайт должен выглядеть современно, но спокойно.

Рекомендуемое направление:

- светлый фон;
- теплые нейтральные оттенки;
- мягкие зеленые, бордовые или золотистые акценты;
- аккуратные карточки кружков и мероприятий;
- понятные кнопки без перегруженности;
- хорошая читаемость текста;
- минимум визуального шума.

Чего лучше избегать:

- слишком ярких кислотных цветов;
- ощущения корпоративной CRM;
- перегруженной админской панели на основных страницах;
- чрезмерно декоративного религиозного оформления;
- темного тяжелого интерфейса.

Подходящее ощущение: теплый приходской сайт, где человеку легко найти кружок, посмотреть расписание и быстро отметить присутствие.

## 9. Что нужно уточнить у заказчика

- Кто именно будет администратором сайта?
- Нужна ли отдельная роль преподавателя?
- Регистрация будет свободной или с подтверждением администратора?
- Можно ли пользователю самостоятельно отписаться от кружка?
- За какое время до занятия можно менять отметку присутствия?
- Нужны ли уведомления о ближайших занятиях?
- Нужно ли хранить историю посещаемости?
- Нужны ли ограничения по возрастным группам?
- Нужно ли показывать мероприятия только будущие или также архив прошедших?

## 10. Минимальная версия проекта

Для первой рабочей версии достаточно реализовать:

- регистрацию и вход;
- роли пользователя и администратора;
- главную страницу с мероприятиями;
- страницу кружков;
- запись пользователя на кружок;
- персональное расписание пользователя;
- отметку "буду" / "не буду";
- административное добавление кружков, мероприятий и занятий;
- просмотр администратором списка присутствующих.

После этого можно добавить календарь на месяц, расширенный профиль, фильтры, уведомления и более подробную статистику.
